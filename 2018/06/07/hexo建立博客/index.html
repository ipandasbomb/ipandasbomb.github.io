hexo建立博客

简要思路：

安装软件建立好ssh后
只需在指定文件夹下打开git bash执行三个命令就可以了
分别是
hexo init #建立项目，然后设置主题，在source目录放入域名
npm install hexo-deployer-git –save # 导入部署
hexo g -d #生成博客并部署到GitHub，需要ssh秘钥。完成。

-①- 建立项目

在GitHub上建立repositry 名字为your_username.github.io

-②- 安装软件

安装好 node.js  和 github for windows

配置ssh

ssh的作用是建立本地和远程库之间连接的管理工具

具体配置参考如下链接：

github设置添加SSH - 破男孩 - 博客园

npm install hexo-cli -g 安装hexo

记录第一次搭建hexo

-③- 建立本地博客

具体内容参考如下链接：
基于Hexo搭建博客并部署到Github Pages
配置好软件环境后，选择自己的博客文件路径，执行以下hexo命令，
如下：
  hexo init             初始化，安装所需包，建立软件库
  npm install           可选操作
  hexo g（generate）     生成
  hexo s（server）       运行
  此时可到localhost：4000 本地预览搭建的博客

  其他相关命令如下：
  hexo n 新建文章，在\source\_posts 文件夹操作
  hexo new page 新建页面，比如在导航栏里新增一个“关于我”的页面
  hexo clean  清除本地生成的public文件夹和数据库
  hexo g 生成博客文件
  hexo s 运行本地浏览器，预览搭建好的博客
  hexo d 部署博客到GitHub上

-④- 本地配置—–部署到GitHub

上步建立了本地博客后，对\blog\hexo 目录的_config.yml文件进行配置
1 首先到GitHub上复制repository 的ssh地址
2 然后到_config.yml上修改。
3 根目录下，执行git命令
  npm install hexo-deployer-git --save
  hexo g
  hexo d
如果没有以上命令，会出现deployer not found 的错误，出现deployer done ：git 说明部署到GitHub成功。失败了可参考如下步骤：
删除  .deploy_git文件夹
       hexo clean
       hexo g
       hexo d
Ø 其他

-⑤- 绑定域名

在\source 文件夹下建立CNAME文件，输入你的域名：xxx.com。无格式保存。然后打开git bash执行命令:
hexo g -d

推送到GitHub。此时会需要ssh的秘钥。
执行后，等待，刷新网站，即可。然后到GitHub项目进行设置，在GitHub pages设置中将域名添加到custom domain中。

-⑥- 域名解析

需要将自己的域名进行解析，这里使用阿里云DNS服务器进行解析。首先命令提示符输入命令 ping GitHub项目名.github.io获得域名ip，然后解析。同时由于我这里是购买的godaddy的域名，使用阿里云解析，那么需要到godaddy修改DNS服务器，修改为阿里云的DNS服务器。具体DNS部分可以参考

-⑦- 主题

在\themes 目录下（打开git bash），执行命令：
git clone xxxxxx（此处为你中意的hexo主题地址）
然后到根目录config文件的theme：xxx（此处填入你拷贝的主题名字）

-⑨- 异地管理博客

► 分支配置
• 可在以上建立的博客的基础上，本地博客不做处理，然后新建一个名为hexo的分支（新建的hexo会有master分支的文档，处理见下），并设置为默认分支 default branch。
• 也可以在建立博客的时候直接建立两个分支，一个为master（存放静态文档），一个为hexo（存放配置文档）
然后在进行配置config.yml 的时候，将branch设置为master。
► 同步：
打开GitHub for Windows clone博客到本地，然后打开博客目录，这时候是在hexo环境，可以打开git bash 查看是hexo环境，保留.git文件夹，其他所有文件删除，复制原先建立的本地博客文件到hexo目录下，然后同步到GitHub上即可。如果换电脑了，直接打开git bash clone博客到本地，然后，执行命令npm install（不然无法形成网页文件），即可开始写博客了。这个方案源码都是公开的。如果注重隐私可采取下方案。


► 或者直接云盘备份本地博客。
